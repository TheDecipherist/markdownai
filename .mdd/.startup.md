## Package Structure
@markdownai/parser -- lexer + AST + directive registry (published standalone)
@markdownai/renderer -- 11 format modules (published standalone)
@markdownai/engine -- template execution, macro expansion, pipe runner, cache
@markdownai/stripper -- removes directives for safe export
@markdownai/mcp -- MCP server, phase management, AI tools
@markdownai/core -- CLI binary `mai`

## Rules Summary
Read CLAUDE.md for the full rulebook. Key rules:
- TypeScript always, strict mode, no `any`
- One directive module per directive -- parser registry pattern (one file per directive in directives/)
- No file > 300 lines, no function > 50 lines
- ESM only -- all imports use .js extensions (resolved to .ts by NodeNext)
- Never commit .env or secrets
- Start with Wave 1 -- never skip ahead
- Always branch -- never commit directly to main
